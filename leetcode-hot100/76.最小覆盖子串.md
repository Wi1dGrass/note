# [76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring/)

给定两个字符串 `s` 和 `t`，返回 `s` 中涵盖 `t` 所有字符（包括重复字符）的最短子串。如果没有则返回空字符串。

## 思路

1. **记录目标频次**：用数组 `need[128]` 记录 `t` 中每个字符需要的数量，用变量 `needCount` 记录还需要匹配的字符总个数
2. **滑动窗口**：维护窗口 `[j, i]`，不断扩张右边界 `i`
   - 右边界字符进入窗口时，`need[c]--`，如果 `need[c] >= 0` 说明匹配了一个有效字符，`needCount--`
   - 当 `needCount == 0` 时窗口已满足条件，尝试收缩左边界 `j` 记录最短子串
3. **记录答案**：用 `ansLeft` 和 `ansRight` 记录最短子串的左右边界

**为什么用 `needCount` 优化**：原做法每次检查窗口是否覆盖都要遍历 52 个字母判断，用计数器只需 O(1) 判断

时间复杂度：O(m + n)
空间复杂度：O(1)，固定大小数组

## 代码

```java
class Solution {
    public String minWindow(String s, String t) {
        char[] cs = s.toCharArray();
        char[] ct = t.toCharArray();

        int[] need = new int[128]; // 记录 t 中每个字符还需要匹配的次数
        for (char c : ct) {
            need[c]++;
        }

        int needCount = ct.length; // 还需要匹配的字符总数
        int ansLeft = -1, ansRight = cs.length; // 记录最短子串边界
        int j = 0; // 左边界

        for (int i = 0; i < cs.length; i++) {
            // 右边界字符进入窗口
            need[cs[i]]--;
            // need >= 0 说明这个字符是 t 需要的，匹配了一个有效字符
            if (need[cs[i]] >= 0) {
                needCount--;
            }

            // 窗口已满足条件，尝试收缩左边界找更短的子串
            while (needCount == 0) {
                // 更新最短子串
                if (i - j < ansRight - ansLeft) {
                    ansLeft = j;
                    ansRight = i;
                }

                // 左边界字符离开窗口
                need[cs[j]]++;
                // need > 0 说明移除的是 t 需要的字符，窗口不再满足
                if (need[cs[j]] > 0) {
                    needCount++;
                }
                j++; // 左边界右移
            }
        }

        return ansLeft < 0 ? "" : s.substring(ansLeft, ansRight + 1);
    }
}
```
