# [41. 缺失的第一个正数](https://leetcode.cn/problems/first-missing-positive/)

给你一个未排序的整数数组 `nums`，找出其中没有出现的最小的正整数。要求时间复杂度 O(n) 且只使用常数级别额外空间。

## 思路

### 方法一：排序（不满足要求）

排序后从 1 开始遍历，找到第一个缺失的正数。但排序需要 O(n log n)，不满足要求。

### 方法二：原地哈希

核心思想：利用数组本身作为哈希表，让 `nums[i]` 的值等于 `i + 1`（即数值 x 放在下标 x-1 的位置）。

1. 遍历数组，如果 `nums[i]` 在 `[1, n]` 范围内且不在正确位置上，就交换到 `nums[nums[i] - 1]` 的位置
2. 再次遍历，第一个 `nums[i] != i + 1` 的位置，缺失的就是 `i + 1`
3. 如果全部匹配，缺失的是 `n + 1`

**为什么不用额外空间？** 直接在原数组上交换，用数组本身存储"数值 x 应该在位置 x-1"的信息

**为什么是 while 不是 if？** 交换过来的新元素也可能需要放到正确的位置，要持续交换直到当前位置正确或无法交换

时间复杂度：O(n)，每个元素最多被交换一次到正确位置
空间复杂度：O(1)

## 代码

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        int n = nums.length;

        // 将每个数放到正确的位置：nums[i] 应该放在下标 nums[i] - 1 的位置
        for (int i = 0; i < n; i++) {
            // 只有当 nums[i] 在 [1, n] 范围内且不在正确位置时才交换
            while (nums[i] >= 1 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                int j = nums[i] - 1; // nums[i] 应该在的下标
                // 交换 nums[i] 和 nums[j]
                int temp = nums[i];
                nums[i] = nums[j];
                nums[j] = temp;
            }
        }

        // 找第一个位置不匹配的元素
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }

        // 全部匹配，缺失的是 n + 1
        return n + 1;
    }
}
```
